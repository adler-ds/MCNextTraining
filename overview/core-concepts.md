# Core Concepts - Marketing Cloud on Core

## Data Model

### Working with Standard Objects
Marketing Cloud on Core uses native Salesforce objects:
- **Leads** - Prospects not yet qualified
- **Contacts** - Known individuals
- **Accounts** - Companies/organizations
- **Opportunities** - Sales deals
- **Campaigns** - Marketing campaigns

No need for subscriber management or data extensions!

## Key Features

### 1. Email Marketing
- Send emails directly from Salesforce
- Use Email Templates and Content Builder
- Track opens, clicks, and engagement
- Native personalization using merge fields

### 2. Journey Builder
- Create automated customer journeys
- Trigger based on Salesforce data changes
- Visual journey canvas in Lightning
- Integration with Flow for complex logic

### 3. Segmentation
- Use Salesforce Reports and List Views
- Dynamic segments based on real-time data
- Standard Salesforce filtering and criteria

### 4. Analytics
- Native Salesforce reports and dashboards
- Track email performance
- Campaign ROI and attribution
- Standard Einstein Analytics integration

### 5. Forms and Landing Pages
- Web-to-Lead forms
- Experience Cloud integration
- Capture data directly into Salesforce objects

## Permissions and Security

Uses standard Salesforce security model:
- Profiles and Permission Sets
- Sharing Rules
- Field-Level Security
- Object-Level Security

## Best Practices

1. **Leverage existing CRM data** - No need to duplicate
2. **Use standard objects** - Easier maintenance and scaling
3. **Follow Salesforce naming conventions** - Consistency across clouds
4. **Design for the platform** - Take advantage of native features
5. **Test in sandbox first** - Standard Salesforce deployment practices
