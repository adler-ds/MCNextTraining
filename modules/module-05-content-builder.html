Understanding Content Builder

Required • 45 minutes

The Five Key Concepts

Content Builder is your one-stop shop for creating all marketing content in MC Next. Let's understand the core building blocks:

1

Email Builder (Drag-and-Drop)

MC Next uses a modern drag-and-drop email builder that doesn't require coding knowledge. You add content blocks (text, images, buttons, spacers) to build your layout.

  • Responsive by default: Emails automatically adapt to mobile/desktop
  • Brand Builder integration: Colors, fonts, logos from your brand kit
  • Personalization: Insert merge fields from Data Graph (name, tier, points)
💡 Key Insight: All emails must reference a Data Graph. The graph determines which personalization fields are available (e.g., {{Individual.FirstName}}, {{LoyaltyMember.Tier}}).
2

Dynamic Content Blocks

Dynamic content lets you show different content to different people within the same email template, based on rules.

Example for NTO:

Rule: If Member Tier = Gold → Show "Premium Trail Access + Free Shipping"
Rule: If Member Tier = Silver → Show "Trail Access + $5 Shipping"
Rule: If Member Tier = Bronze → Show "Trial Trail Access Available"

One email template, three experiences!

3

Brand Builder & Template Library

Brand Builder: Centrally manage NTO's brand assets (logo, colors, fonts) so all emails look consistent.

Template Library: Save email layouts as templates for reuse. NTO has:

  • NTO_Welcome_Template: For new member onboarding
  • NTO_Trail_Promo: For engagement trail invitations
  • NTO_Transactional: For order confirmations and receipts
💡 Best Practice: Use naming conventions like NTO_[Type]_[Purpose]_v[#] to keep your content library organized.
4

Forms & Landing Pages

MC Next includes a form builder for data capture and preference management.

Use cases for NTO:

  • Preference Center: Let members choose email, SMS, WhatsApp opt-ins
  • Event Registration: Sign up for trail challenges
  • Profile Update: Change email address or phone number

Forms are embedded in landing pages with NTO branding. When a member submits, data flows back to Data Cloud and updates their Individual record.

5

Reusable Content Blocks

Create content blocks once and reuse them across multiple emails. Common blocks for NTO:

  • Header block: NTO logo + navigation
  • Footer block: Social links, address, unsubscribe link (required!)
  • Tier benefits block: Dynamic content showing benefits by tier
  • CTA block: Branded "Shop Now" or "Join Trail" button
💡 Pro Tip: If you update a reusable block, all emails using it get the update. Perfect for changing a promo or fixing a typo!

Create Welcome Email Template

Hands-on • 45 minutes

🎯 Your Mission

Build a branded welcome email for new NTO Rewards members with tier-based personalization. Sarah Johnson (Silver tier) and Marcus Chen (Gold tier) should see different benefit messaging.

✅ Success Criteria:
  • Create email template with NTO branding (colors, logo, fonts)
  • Add personalization merge fields (FirstName, Tier, PointsBalance)
  • Build dynamic content block showing tier-specific benefits
  • Preview for Bronze, Silver, and Gold tier members

📋 Step-by-Step Instructions

1

Navigate to Content Builder

From the MC Next app, click Content in the left navigation.

Click Create → Email.

Select Start from Scratch (we'll build custom layout).

Content menu showing Email creation option

2

Set Up Email with Data Graph

You'll enter the Email Builder interface. First, configure the basics:

  • Email Name: NTO_Welcome_Email_v1
  • Subject Line: Welcome to NTO Rewards, {{Individual.FirstName}}!
  • From Name: NTO Rewards Team
  • From Email: rewards@ntomail.com
  • Data Graph: Select NTO_Rewards_Personalization (the graph you built in Module 3)

Email Builder with Data Graph selection

📸 mod5-step2-email-builder.jpg

💡 Pro Tip: Choosing the Data Graph now makes those fields available for merge fields throughout the email.
3

Build Email Layout with NTO Branding

Now drag and drop content blocks to build the email:

  1. Header Image: Drag an Image block. Upload NTO logo (or use from Brand Builder assets).
  2. Hero Text: Drag a Text block. Add: "Welcome to NTO Rewards, {{Individual.FirstName}}!"
  3. Body Paragraph: Add another Text block with welcome message.
  4. CTA Button: Drag a Button block. Text: "Explore Your Benefits". Link to: https://www.nto.com/rewards-portal
  5. Footer: Add social icons, address, and unsubscribe link (required by law!).

Apply NTO brand colors:

  • Select the hero text, change color to NTO Blue (#0176D3)
  • Select the button, change background to NTO Orange (#FF6B35)

Complete email layout with header, body, CTA, and footer

Add Personalization Merge Fields

Let's make this email personal! Click into any text block and click the Personalization icon (looks like {})

You'll see all fields from your Data Graph. Add these:

  • First Name: {{Individual.FirstName}} (already done in hero)
  • Member Tier: Add a line: "You're a {{LoyaltyMember.Tier}} member!"
  • Points Balance: Add: "Current Points: {{LoyaltyMember.PointsBalance}}"

Inserting personalization tokens from NTO Rewards Data Graph

Add Dynamic Content for Tier-Based Promotions

Hands-on • 45 minutes

🎯 Your Mission

Enhance the welcome email with a dynamic content block that shows different engagement trail promotions based on member tier. Gold members see premium trails, Silver sees standard trails, Bronze sees trial access.

✅ Success Criteria:
  • Create dynamic content block with 3 tier variations

  • Silver tier: Show "Valley Explorer Trail" promo

  • Preview email as each tier to verify content switches

📋 Step-by-Step Instructions

1

Insert Dynamic Content Block

In your email (from Lab 1), add a new section below the welcome text.

From the content blocks panel, drag in a Dynamic Content block.

You'll be prompted to set up rules. Click Create Rules.

Configure Tier-Based Rules

Set up three content variations based on LoyaltyMember.Tier:

As a Gold member, you have access to our most challenging trails with premium rewards.
Complete 10 activities in 30 days and earn 5,000 bonus points + free shipping for life!
[Join Summit Seeker Trail →]

Rule 2: Silver Tier

  • Condition: LoyaltyMember.Tier equals "Silver"
  • Content:
🌲 Your Trail: Valley Explorer
Perfect for intermediate adventurers! Complete 7 activities in 30 days.
Earn 2,000 bonus points + $5 shipping on your next order.
[Join Valley Explorer Trail →]

  • Condition: Default (everyone else)

[Try Your First Trail →]

📋 Step-by-Step Instructions

1

Create Preference Center Form

From Content Builder, click Create → Form.

Name: NTO_Preference_Center_Form

Drag the following fields from the left panel:

Preference center form with multi-channel consent options

Map form fields to Data Cloud:

  • Email checkbox → Individual.HasOptedOutOfEmail (inverse logic)

2

Build Landing Page

Now create a landing page to host this form.

Click Create → Landing Page.

Name: NTO_Preference_Center

Use a simple layout:

  • Header: NTO logo + "Communication Preferences"
  • Body text: "Manage how you hear from NTO Rewards"

Deep Dive: AI Features & Approval Workflows

Optional • 30 minutes

Einstein AI Features in Content Builder

MC Next includes several AI-powered features to optimize content performance:

Send Time Optimization (STO)

Einstein analyzes each recipient's engagement history and sends at their optimal time.

Use when: You have at least 3 months of email history.

Benefit: 15-20% lift in open rates for NTO campaigns.

Einstein Metrics Guard

AI predicts email performance BEFORE you send. Warns if subject line will underperform.

Example: "Subject too long (150 chars) - consider shortening to 50."

Benefit: Catch mistakes before they impact deliverability.

Generative AI Content Assistant

Generate subject lines, body copy, and CTAs using AI prompts.

Example prompt: "Write a welcome email for new Gold tier loyalty members, highlighting exclusive benefits."

Note: Always review and edit AI-generated content!

Content Approval Workflows

For regulated industries or large teams, MC Next supports approval workflows using CMS (Content Management System) roles:

  • Content Creator: Builds emails, marks "Ready for Review"
  • Content Reviewer: Approves or rejects with feedback
  • Content Publisher: Final publish after approval

NTO use case: Legal team must approve all loyalty program terms mentioned in emails.

Template Governance & Naming Conventions

As your content library grows, governance becomes critical. Best practices:

  • Naming: [Brand]_[Type]_[Purpose]_v[#] → NTO_Email_Welcome_v2
  • Folders: Organize by campaign or lifecycle stage (Acquisition, Onboarding, Retention)
  • Version control: Always increment version number, never overwrite live templates
  • Archive old content: Move unused templates to "Archive" folder after 6 months

Accessibility Best Practices

Ensure your emails are accessible to all members:

  • Alt text: Add descriptive alt text to all images ("NTO logo" not "image1.jpg")
  • Color contrast: Ensure text has sufficient contrast ratio (4.5:1 minimum)
  • Link text: Use descriptive links ("View your rewards" not "Click here")
  • Semantic HTML: Use proper heading structure (H1, H2, H3)

🏗️ IMPLEMENTATION NOTES

For Implementation Teams

Optional • 15 minutes

Discovery Questions for Customers

When scoping content requirements, ask:

Q: Do you have existing brand guidelines (logo, colors, fonts)?

Why it matters: Determines whether to use Brand Builder or custom HTML.

Q: How many email templates do you currently use?

Why it matters: Migration effort (rebuilding templates in MC Next).

Q: Do you need content approval workflows?

Why it matters: Requires CMS role setup and training.

Q: What personalization fields are most important?

Why it matters: Determines Data Graph structure.

Template Migration Strategy

If migrating from Marketing Cloud Classic or another ESP:

  • Audit existing templates: Catalog all active templates (often 50-100+)
  • Prioritize: Rebuild high-volume templates first (welcome series, newsletters)
  • Test personalization: Verify all merge fields work in new Data Graph structure
  • Parallel run: Keep old system running during migration for validation

Performance Optimization

Optimization Impact Implementation
Image compression Faster load times Keep images under 200KB, use WebP format
Reusable content blocks Faster template creation Build header/footer blocks once, reuse everywhere
Dynamic content caching Faster send times Pre-render common variations (tier-based content)

Common Pitfalls & Solutions

⚠️

Pitfall: Forgetting unsubscribe link

Symptom: Email fails send validation

Solution: Always include unsubscribe link in footer (required by CAN-SPAM).

⚠️

Pitfall: Dynamic content rules overlap

Symptom: Wrong content shown to some recipients

Solution: Use mutually exclusive conditions and always have a Default rule.

⚠️

Pitfall: Preference center not updating Data Cloud

Symptom: Consent changes don't suppress sends

Solution: Verify form fields map correctly to ContactPointConsent DMO.

When to Escalate

  • Custom HTML needs: If brand requires complex layouts beyond drag-and-drop, engage front-end developer
  • Multi-language content: If global campaigns need translation, engage localization team
  • Complex approval workflows: If more than 2 approval steps needed, engage admin for custom workflow
