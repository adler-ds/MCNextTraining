📖 CORE LESSON

Understanding Flow Builder

Required • 45 minutes

The Four Key Concepts

Flow Builder is MC Next's journey automation engine. Let's understand the building blocks:

1

Trigger Types

Every flow starts with a trigger that determines how members enter the journey:

  • Segment-triggered: Members enter when they join a segment (e.g., "New Members" segment)
  • Event-triggered: Members enter when a specific event occurs (e.g., "Trail Started" event from Data Cloud)
  • On-Demand: Manually trigger for a specific audience (e.g., flash sale announcement)
💡 Key Insight: For NTO's welcome journey, use segment-triggered. For trail promotions, use event-triggered to respond in real-time to member actions.
2

Interaction Elements

Interactions are the touchpoints where you communicate with members:

Available channels:

Email: Best for detailed content, product showcases, educational series
SMS: Urgent alerts, time-sensitive promotions, appointment reminders
WhatsApp: Two-way conversations, customer service escalations
Agent Handoff: Transfer to Service Cloud when journey needs human intervention

Each interaction element references content you've created in Content Builder.

3

Wait & Decision Splits

Wait Elements: Control timing between interactions

  • Wait for Duration: Fixed time (e.g., "Wait 2 days")
  • Wait Until Date: Specific date/time (e.g., "Wait until 10am Pacific")
  • Wait for Event: Wait until member performs action (e.g., "Wait for Purchase" up to 7 days)

Decision Splits: Branch based on conditions

  • Example: If LoyaltyMember.Tier = Gold → Send SMS + Email, else → Send Email only
  • Use for: Tier-based experiences, engagement-based paths, purchase behavior
💡 Best Practice: Use "Wait for Event" with a timeout. Example: "Wait for first purchase (max 7 days). If purchased → Send thank you. If timeout → Send incentive."
4

Agent Handoff

When automation isn't enough, hand off to Service Cloud for human intervention:

Use cases for NTO:

  • High-value tier escalation: Gold member has issue → Create case for VIP support team
  • Journey abandonment: Member doesn't engage after 3 emails → Trigger outreach call
  • Complex questions: Member replies to WhatsApp message → Route to agent

Agent Handoff creates a Service Cloud case with full journey context (which emails sent, clicks, opens).

Build Welcome Journey

Hands-on • 60 minutes

🎯 Your Mission

Create a 3-email welcome series for new NTO Rewards members. When Sarah Johnson joins the "New Members" segment, she should receive: Day 0 Welcome, Day 2 Getting Started, Day 7 First Purchase Incentive.

✅ Success Criteria:
  • Create segment-triggered flow from "New Members" segment
  • Add 3 email interactions with appropriate Wait elements
  • Use emails created in Module 5 (or templates)
  • Activate flow and verify members can enter

1

Navigate to Flow Builder

From the MC Next app, click Journeys in the left navigation.

Click Create Journey → Segment-triggered.

Name: NTO_Welcome_Journey_v1

Select segment: New_Members (the segment you built in Module 2)

Implement A/B Testing with Path Experimentation

Hands-on • 60 minutes

🎯 Your Mission

Test two different subject lines for the trail invitation email to see which drives more opens. Use Path Experimentation to split traffic 50/50 and measure performance.

✅ Success Criteria:
  • Add Path Experimentation element to existing flow
  • Create two paths with different subject lines
  • Set 50/50 traffic split
  • Define success metric (email opens)
  • Activate test and verify split is working

📋 Step-by-Step Instructions

1

Add Path Experimentation Element

Open your Trail Promotion flow from Lab 2.

Before the Decision Split, insert a Path Experimentation element.

Configure:

  • Experiment Name: Trail Invitation Subject Line Test
  • Number of Paths: 2 (Variant A and Variant B)
  • Traffic Split: 50% / 50%

A/B test element with 50/50 split

📸 mod6-lab3-step1-experimentation.jpg

💡 Pro Tip: MC Next automatically randomizes members into paths and tracks performance for you!
2

Configure Variant A: Benefit-Focused Subject

On Path A, add an Email interaction:

  • Email Name: Trail Invite - Variant A
  • Subject Line A: Earn 5,000 bonus points on the Summit Seeker Trail!
  • Content: Standard trail invitation email (same for both variants)

Variant A emphasizes point reward

📸 mod6-lab3-step2-variant-a.jpg

3

Configure Variant B: Urgency-Focused Subject

On Path B, add an Email interaction:

  • Email Name: Trail Invite - Variant B
  • Subject Line B: Only 3 days left to join the Summit Seeker Trail
  • Content: Same email content as Variant A

Variant B emphasizes urgency

Set Success Metric

Deep Dive: Multi-Channel Orchestration & Error Handling

Optional • 30 minutes

Profile Selection for Multi-Brand Flows

If you're managing multiple brands (e.g., NTO Retail vs NTO Rewards), use profile selection:

  • Profile Selection element: Route members based on Brand preference
  • Example: If Brand = "NTO Retail" → Retail welcome journey, else → Rewards welcome journey
  • Benefit: One entry trigger, multiple brand-specific experiences

Error Handling & Flow Monitoring

Production flows need robust error handling:

Email Send Failures

What if an email bounces or member has no email address?

Solution: Add fallback path. If email fails → Send SMS (if available) or create Service Cloud case.

Event Data Missing

What if "Trail Name" field is missing in event payload?

Solution: Use default value. Example: {{Trail.Name | default: "an engagement trail"}}

Journey Monitoring

How do you know if a flow is broken?

Solution: Set up Journey Health alerts. Get notified if error rate exceeds 5%.

Multi-Channel Orchestration Best Practices

When mixing Email, SMS, and WhatsApp in one journey:

  • Check consent: Add decision split to verify member has opted in to SMS/WhatsApp before sending
  • Respect frequency: Don't send SMS and Email in same hour. Add 2-hour wait between channels.
  • Channel hierarchy: Use SMS for urgent/transactional, Email for promotional/educational
  • WhatsApp for two-way: Only use WhatsApp if you have agents available to respond

Advanced Wait Strategies

Beyond simple duration waits:

  • Einstein Send Time Optimization: Wait until optimal send time for each member (ML-powered)
  • Wait until attribute change: Wait until LoyaltyMember.Tier changes (max 90 days)
  • Business hours wait: Wait until 9am-5pm Pacific (don't send overnight)
  • Event-based wait: Wait for "Purchase" event (timeout 14 days)

🏗️ IMPLEMENTATION NOTES

For Implementation Teams

Optional • 15 minutes

Naming Conventions & Governance

Establish flow naming standards early:

  • Format: [Brand]_[Type]_[Purpose]_v[#]
  • Example: NTO_Welcome_Journey_v2, NTO_Abandoned_Cart_v1
  • Never overwrite: Always version up (v1 → v2) and deactivate old version
  • Folder structure: Organize by lifecycle stage (Acquisition, Onboarding, Engagement, Retention)

Testing Strategy Before Production

Always test flows in sandbox before activating in production:

Test Type What to Check How
Validation Test No errors in flow logic Click "Review" button - fix any errors
Data Test Personalization renders correctly Use "Preview" with real member data
Timing Test Waits execute as expected Activate with test segment, monitor for 24 hours
Multi-path Test Decision splits route correctly Test with members in each tier/segment

Performance Monitoring in Production

Key metrics to track for each active flow:

  • Entry Rate: How many members entering per day? (Should match segment size or event volume)
  • Completion Rate: % of members who reach the end of journey
  • Drop-off Points: Where are members exiting early? (Indicates disengagement)
  • Error Rate: % of interactions that fail (email bounce, SMS error, etc.)
  • Engagement Rate: Opens, clicks, conversions per journey

Common Pitfalls & Solutions

⚠️

Pitfall: Members enter journey multiple times

Symptom: Sarah gets the welcome series twice

Solution: Set entry criteria: "Has not entered this journey before" or use "Entry limit: 1 time"

⚠️

Pitfall: Event-triggered flow fires too frequently

Symptom: Member starts 5 trails in 1 day → Gets 5 separate emails

Solution: Add "Frequency Cap: Max 1 entry per member per day"

⚠️

Pitfall: Personalization field missing

Symptom: Email shows "Hello, {{Individual.FirstName}}" instead of "Hello, Sarah"

Solution: Use default values: {{Individual.FirstName | default: "Valued Member"}}

When to Escalate

  • Complex business logic: If decision splits require 5+ conditions, consider splitting into multiple flows
  • Custom events: If needed event doesn't exist in Data Cloud, engage data team to create streaming event
  • Agent handoff setup: Requires Service Cloud configuration - engage Service Cloud admin
  • High-volume performance: If flow will send 1M+ messages/day, engage platform architect for capacity planning
