Understanding Analytics & Reporting

Required • 30 minutes

The Four Key Concepts

MC Next provides multiple layers of analytics to measure campaign and program performance. Let's understand the core building blocks:

1

Micro Dashboards (Campaign-Level)

Micro dashboards show performance for individual campaigns and journeys:

  • Journey Analytics: View performance of a specific flow (e.g., Welcome Journey)
  • Metrics tracked: Sends, Opens, Clicks, Bounces, Unsubscribes, Conversions
  • Drill-down views: By device (mobile vs desktop), location, time of day
  • Path analysis: See where members drop off in journey
💡 Use Case: NTO's marketing manager checks the Welcome Journey micro dashboard daily to optimize timing and content. If Day 2 email has low opens, test a different subject line.
2

Macro Dashboards (Strategic Planning)

Macro dashboards provide high-level insights across all campaigns:

Marketing Performance Dashboard shows:

Global metrics: Total sends, average open rate, click-through rate across all campaigns
Trend analysis: Are metrics improving month-over-month?
Channel comparison: Email vs SMS vs WhatsApp performance
Segment performance: Gold members vs Silver vs Bronze engagement

Purpose: Strategic planning, executive reporting, identifying patterns.

3

Semantic Data Model (Raw Data Exports)

For advanced analysis, export raw data using the Semantic Data Model:

  • What it is: Direct access to underlying campaign data (sends, opens, clicks) as SQL tables
  • Export to: Tableau, CRM Analytics, Excel, external BI tools
  • Use cases: Custom reports, complex attribution modeling, cross-system analysis

Example query: "Show me all members who opened 3+ welcome emails but never made a purchase."

💡 Pro Tip: Use built-in dashboards for day-to-day monitoring. Export raw data for deep-dive analysis or custom reports.
4

Loyalty Lift (ROI Calculation)

The ultimate question: Does the loyalty program drive more business value?

Loyalty Lift measures:

  • Purchase frequency: Do loyalty members buy more often than non-members?
  • Average order value: Do members spend more per transaction?
  • Lifetime value: What's the total value of a member over 3 years vs non-member?
  • Retention rate: Do members churn less than non-members?

NTO Example:

Loyalty Members: Purchase 4.2 times/year, AOV $95
Non-Members: Purchase 1.8 times/year, AOV $75
Loyalty Lift: 2.3x purchase frequency, 27% higher AOV
Result: Loyalty program ROI = 320% (members generate 3.2x more revenue)

Calculate Loyalty Metrics & ROI

Hands-on • 45 minutes

🎯 Your Mission

Build a custom report showing loyalty-specific metrics: enrollment rate per month, tier progression tracking, and trail completion percentage. Then calculate loyalty lift ROI by comparing member vs non-member purchase behavior.

✅ Success Criteria:
  • Create custom report for enrollment rate trend
  • Track tier progression (Bronze→Silver→Gold) over time
  • Calculate trail completion percentage
  • Calculate loyalty lift: purchase frequency members vs non-members
  • Export report for executive presentation

📋 Step-by-Step Instructions

1

Create Enrollment Rate Report

Go to Analytics → Custom Reports → Create Report.

Name: NTO_Loyalty_Enrollment_Trend

Configure the report:

  • Data Source: LoyaltyProgramMember
  • Metric: Count of new members (where EnrollmentDate >= FirstDayOfMonth)
  • Dimension: Month
  • Date Range: Last 12 months

You'll see a trend line showing:

  • January: 4,200 new enrollments
  • February: 4,800 new enrollments
  • March: 5,500 new enrollments (growing!)

Monthly new member enrollment showing 12-month trend

📈 Insight: Enrollment is growing 10-15% month-over-month. Correlate with marketing campaigns to identify which channels drive enrollments.
2

Track Tier Progression

Create a second report: NTO_Tier_Progression

Configure:

  • Data Source: LoyaltyMemberTier with date filter on EffectiveDate
  • Metric: Count of tier changes
  • Dimensions: Previous Tier, New Tier, Month

Results for March:

  • Bronze → Silver: 1,800 members upgraded
  • Silver → Gold: 450 members upgraded
  • Total Active: Bronze: 45K, Silver: 18K, Gold: 5K

Monthly tier upgrades showing progression funnel

📸 mod8-lab2-step2-tier-progression.jpg

💡 Insight: 4% of Bronze members upgrade to Silver each month. 2.5% of Silver upgrade to Gold. These are healthy progression rates indicating engaged member base.
3

Calculate Trail Completion Rate

Create report: NTO_Trail_Completion_Rate

Query logic:

  • Started: Count of Promotion_Joined events
  • Completed: Count of Promotion_Completed events
  • Completion Rate: (Completed / Started) * 100

Results by trail:

  • Beginner's Adventure: 78% completion (easy trail, high success)
  • Valley Explorer: 62% completion
  • Summit Seeker: 45% completion (challenging, but engaged Gold members)
📈 Insight: Summit Seeker has 45% completion. That's actually good for a difficult trail! Consider adding mid-journey encouragement email to boost to 50%+.
4

Calculate Loyalty Lift ROI

This is the big one! Calculate the business value of the loyalty program.

Go to Data Cloud → Query → Create Query.

Name: Loyalty_Lift_Analysis

Query 1: Member Purchase Behavior

SELECT
  COUNT(DISTINCT Order.Id) / COUNT(DISTINCT Individual.Id) AS PurchasesPerMember,
  AVG(Order.TotalAmount) AS AverageOrderValue,
  SUM(Order.TotalAmount) / COUNT(DISTINCT Individual.Id) AS LifetimeValue
FROM Individual
JOIN LoyaltyProgramMember ON Individual.Id = LoyaltyProgramMember.ContactId
JOIN Order ON Individual.Id = Order.CustomerId
WHERE Order.OrderDate >= '2025-01-01'

Results for Loyalty Members:

  • Purchases per member: 4.2 per year
  • Average order value: $95
  • Lifetime value (12 months): $399

Query 2: Non-Member Purchase Behavior (same query, exclude LoyaltyProgramMember join)

  • Purchases per customer: 1.8 per year
  • Average order value: $75
  • Lifetime value (12 months): $135

Calculate Loyalty Lift:

  • Purchase Frequency Lift: 4.2 / 1.8 = 2.33x (members buy 233% more often!)
  • AOV Lift: $95 / $75 = 1.27x (members spend 27% more per order)
  • LTV Lift: $399 / $135 = 2.96x (members generate 3x more revenue)

Deep Dive: CRM Analytics & Data Cloud Insights

Optional • 30 minutes

CRM Analytics Dashboards

For enterprise customers, CRM Analytics (formerly Tableau CRM) provides advanced visualization:

  • Interactive dashboards: Drill-down, filter, explore data in real-time
  • Predictive analytics: Einstein forecasts future enrollment, churn risk, LTV
  • Custom calculations: Build complex metrics (e.g., cohort analysis, attribution modeling)
  • Mobile-optimized: View dashboards on mobile for on-the-go insights

Data Cloud Insights

Data Cloud provides AI-powered insights automatically:

Anomaly Detection

Einstein alerts you if metrics suddenly drop (e.g., open rate drops from 28% to 15%).

Benefit: Catch deliverability issues before they impact full campaign.

Trend Forecasting

Predict future enrollment based on historical trends.

Use case: "At current growth rate, NTO will hit 100K members by Q4."

Segment Insights

Einstein identifies which segments are most valuable.

Example: "Gold members in Mountain region have 2x higher LTV."

Tableau Integration

For customers with Tableau licenses, connect MC Next data to Tableau Desktop:

  • Data connector: Use Salesforce Data Cloud connector in Tableau
  • Pre-built dashboards: Salesforce provides loyalty dashboard templates
  • Custom visualizations: Build executive dashboards with brand colors, logos
  • Combine data sources: Merge MC Next data with external data (e.g., Google Analytics, CRM)

Attribution Modeling

Advanced question: Which touchpoints drive conversions?

  • First-touch attribution: Credit to first email member opened
  • Last-touch attribution: Credit to last email before purchase
  • Multi-touch attribution: Credit distributed across all touchpoints

NTO Example: Sarah's journey: Welcome Email (Day 0) → Getting Started (Day 2) → First Purchase Email (Day 7) → Purchase. Multi-touch model gives 40% credit to each email.

🏗️ IMPLEMENTATION NOTES

For Implementation Teams

Optional • 15 minutes

Reporting Governance

Establish governance before rolling out analytics to teams:

  • Single source of truth: Designate Marketing Performance as the official dashboard
  • Naming conventions: [Brand]_[Metric]_[Timeframe] (e.g., NTO_Enrollment_12Mo)
  • Access control: Campaign managers see their campaigns, leadership sees global view
  • Data definitions: Document how metrics are calculated (e.g., "Open rate = Unique opens / Delivered sends")

Scheduled Exports & Distribution

Automate reporting to reduce manual work:

Report Frequency Audience
Campaign Performance Summary Weekly (Monday 8am) Marketing team
Loyalty Lift ROI Monthly (1st of month) Leadership + finance
Journey Health Alerts Daily Campaign managers (only if errors)
Tier Progression Report Monthly Loyalty program manager

Dashboard Refresh Schedules

Configure how often dashboards update:

  • Real-time dashboards: Journey Health, Campaign Performance (15-minute refresh)
  • Daily dashboards: Segment performance, engagement metrics (6am daily)
  • Weekly dashboards: Strategic planning views (Sunday night)
  • Monthly dashboards: ROI reports, executive summaries (1st of month)

Metrics Validation Process

Before sharing reports with executives, validate metrics:

Q: Do send counts match what's in journey logs?

Check: Compare dashboard "Total Sends" to journey entry count.

Q: Are open rates reasonable (not 0% or 100%)?

Check: Industry benchmark is 15-30% for retail. If you're way off, investigate.

Q: Does loyalty lift make business sense?

Check: 2-3x LTV lift is typical. If showing 10x, likely data error.

Q: Are date ranges consistent across reports?

Check: Don't compare Q1 data to Q4 - seasonality skews results.

Common Pitfalls & Solutions

⚠️

Pitfall: Open rates look inflated

Symptom: 60%+ open rates (too good to be true)

Solution: Check if Apple Mail Privacy Protection is counted. Exclude machine opens for accurate metrics.

⚠️

Pitfall: Loyalty lift calculation excludes inactive members

Symptom: ROI looks too high

Solution: Include ALL members (active + inactive) in denominator for accurate LTV calculation.

⚠️

Pitfall: Dashboard shows different numbers than email

Symptom: Team loses trust in reporting

Solution: Check refresh schedules. Dashboard may be cached. Add "Last Updated" timestamp to all dashboards.

When to Escalate

  • Custom attribution modeling: Requires Data Cloud expert to build multi-touch model
  • CRM Analytics setup: Engage CRM Analytics consultant for dashboard development
  • Data discrepancies: If metrics don't match across systems (MC Next vs Data Cloud vs CRM), engage data architect
  • Tableau integration: Requires Tableau Desktop license + data connector setup
